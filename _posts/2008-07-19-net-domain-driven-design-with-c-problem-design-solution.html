Since i didn't find many reviews on Tim McCarthy's</a> book: .NET Domain-Driven Design with C#: Problem - Design - Solution</a>, i've decided to write a short one myself:</p>

This book offers a real-world example of a project using DDD</a>. If you already have experience with DDD you will read pretty quickly through the chapters.</p>

Here are a couple of items in the book i found noteworthy:</p>

  • The IAggregateRoot marker interface that is used as a constraint on the IRepository interface.</li>
  • Code that demonstrated the power of WPF through the implementation of Model-View-ViewModel</a> and usage of the ICommand</a>.</li>
  • Used the IUnitOfWork.Commit</a> method to plug a Synchronization Service into the application.</li>
    </ul>

    All in all, the book is worth the 26$. In case that you're looking for more examples, i would recommend Domain Driven Design - Table of contents and source code</a> as another source of inspiration...</p>
