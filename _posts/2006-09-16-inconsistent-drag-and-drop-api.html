Every System.Windows.Forms.Control has the following events:</p>

  • public event DragEventHandler DragDrop</a></li>
  • public event DragEventHandler DragEnter</a></li>
  • public event EventHandler DragLeave</a></li>
  • public event DragEventHandler DragOver</a></li>
    </ul>

    This means, when the user drags something away of the control, you can't access the data anymore (unless you cached it somewhere when the DragEnter or DragOver events occured). Here is an extract from Microsoft patterns and practives on Event Design</a>:</p>

    If you define an event that takes an EventArgs instance instead of a derived class that you define, you cannot add data to the event in later versions. For that reason, it is preferable to create an empty derived class of EventArgs. This allows you add data to the event in later versions without introducing breaking changes.
    </div>
    </blockquote>
