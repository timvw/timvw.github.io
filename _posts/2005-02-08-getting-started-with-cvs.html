I got an e-mail that asked me how to get started with CVS as quick as possible. The first time i got lost too. So i'll give a quick summary how i did it (no p-server).</p>

The repository will live on a debian</a> machine in /home/users/timvw/services/cvs.</p>

[code lang="bash"]
timvw@debian: apt-get install cvs # get the tools
timvw@debian: cvs -d /home/users/timvw/services/cvs init # create repository
[/code]

On my windows machine i use TortoiseCVS</a>. I want to make a module pecl that will contain all my pecl</a> related code.</p>

1. Configure TortoiseCVS to use the SSH.com</a> client instead of the built-in Putty</a>.</p>

</p>

2. Fill in the path to the SSH binary (tools tab).</p>

</p>

3. Create a directory that will hold the code (i choose pecl as name).</p>

</p>

4. Create a module (Choose the directory you created and click right).</p>

</p>

5. Fill in the settings.</p>

</p>

6. Hit Ok to initialise the module.</p>

</p>

7. Create a README file and add it to the module.</p>

</p>

</p>

8. Add a comment message</p>

</p>

9. Commit the changes</p>

</p>

10. Update your files (Other people have commited their changes).</p>

</p>

11. Read The Fine Manual at http://cvsbook.red-bean.com/cvsbook.html</a></p>
