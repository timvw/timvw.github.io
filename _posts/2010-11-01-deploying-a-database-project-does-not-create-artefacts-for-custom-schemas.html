Last week we noticed that the artefacts for a custom schema in our Database Project were not created (or updated). It took us a while to figure out what was wrong. When you add a script via the 'Add Existing item' menu in VS2010 the file's Build Action</b> is set to Not in Build</b> for some unknown reason. As soon as we changed the value to Build</b> and Deployed again our artefacts were created.</p>
