A while ago i was listening to 'Kathleen Dollard on MEF and More' on NET Rocks!</a> and it got me thinking.</p>

For a long time i considered code generation as a way to reduce the tedious parts of software development by spitting out considerable amounts of code and I immediately thought of tools like T4</a>: Text Template Transformation Toolkit and GAT</a>: Guidance Automation Toolkit.</p>

I believe that in order to become more productive we need a solution that requires less code spitting by means of better frameworks and higher languages. Eg: C# allows us to think about computer instructions on a much higher level than assembly language.</p>

And then i realised that the C# compiler is nothing more than a code generation tool!</p>
