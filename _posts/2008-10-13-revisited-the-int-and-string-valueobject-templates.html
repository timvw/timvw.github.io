After reading The Compare Contract</a> last week i realized that my templates for int and string ValueObjects</a> did not comply with the contract so i decided to add a unittest that reproduces the faulty behavior (and then corrected the implementation):</p>

[code lang="csharp"]///

 /// When comparing instance with null, instance should be greater.
/// </summary>
[TestMethod]
public void ShouldReturnPositiveWhenComparedWithNull()
{
$classname$ value = new $classname$("0");
Assert.IsTrue(value.CompareTo(null) > 0);
}[/code]

Anyway, feel free to download the corrected IntValueObject</a> and StringValueObject</a> templates.</p>
