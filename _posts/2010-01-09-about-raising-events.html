Very often i see people write the following to 'safely' raise a method:</p>

[code lang="csharp"]public event EventHandler Stopped;

void RaiseStoppedEvent()
{
if (Stopped != null) Stopped(this, EventArgs.Empty);
}[/code]

Some developers think that they should program defensively and avoid the potential concurrency problem:</p>

[code lang="csharp"]public event EventHandler Stopped;

void RaiseStoppedEvent()
{
var handler = Stopped;
if (handler!= null) handler(this, EventArgs.Empty);
}[/code]

And then there is Tim's way to raise an event: (If i'm not mistaken it was Ayende</a> who once blogged about this) </p>

[code lang="csharp"]public event EventHandler Stopped = delegate { };

void RaiseStoppedEvent()
{
Stopped(this, EventArgs.Empty);
}[/code]
